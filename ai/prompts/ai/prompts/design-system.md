# CORE PRINCIPLES

## 1. CLARITY FIRST

The user must instantly understand:
- what this is
- what matters
- what to do next

Reduce cognitive load relentlessly.

Avoid confusion at all costs.

---

## 2. HIERARCHY IS EVERYTHING

Use:
- spacing
- typography
- contrast
- scale
- layout
- alignment

to guide attention naturally.

Important things must feel important.

---

## 3. DESIGN MUST BREATHE

Use generous whitespace.

Avoid crowded layouts.

Premium interfaces feel calm, structured, and readable.

---

## 4. RESTRAINT CREATES QUALITY

Avoid:
- visual noise
- excessive colors
- unnecessary borders
- too many effects
- random gradients
- over-designed UI

Every element must have purpose.

---

## 5. CONSISTENCY BUILDS TRUST

Maintain consistency across:
- spacing
- typography
- buttons
- cards
- colors
- interactions
- border radius
- shadows

The interface should feel like one unified system.

---

## 6. MODERN PRODUCT THINKING

Do not design isolated sections.

Design complete experiences.

Everything should flow naturally:
- layout
- interactions
- navigation
- content
- actions

---

## 7. MOTION SHOULD FEEL PREMIUM

Animations must be:
- subtle
- smooth
- intentional
- functional

Motion should improve:
- continuity
- hierarchy
- responsiveness
- perceived quality

Avoid distracting animations.

---

# UI RULES

## Typography
- Strong hierarchy
- Short readable text
- Balanced line lengths
- Clear contrast
- Large confident headlines

---

## Spacing
- Use consistent spacing systems
- Create visual rhythm
- Allow sections to breathe
- Separate content clearly

---

## Colors
- Use restrained color palettes
- Prefer neutral foundations
- Use accent colors intentionally
- Color should guide attention

---

## Components
- Components should feel reusable
- Keep layouts clean and scalable
- Prioritize usability over decoration

---

# UX RULES

Always optimize for:
- clarity
- speed
- simplicity
- confidence
- intuitive navigation

The interface should feel obvious.

---

# CONVERSION RULES

Every page should:
- build trust
- reduce hesitation
- guide progression
- reinforce value
- make actions feel natural

CTAs must be obvious and intentional.

---

# ANTI-PATTERNS

Avoid:
- template-looking layouts
- weak hierarchy
- cluttered sections
- inconsistent spacing
- excessive text
- random UI styles
- flat experiences
- generic dashboards
- aggressive animations
- poor mobile responsiveness

---

# QUALITY STANDARD

The final result should feel:
- premium
- polished
- production-ready
- modern
- scalable
- emotionally intentional

Every pixel must feel designed, not assembled.

If the interface does not feel comparable to modern top-tier products,
refine it again.
